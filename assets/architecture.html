Slurm-on-K8s — System Architecture

Elastic HPC scheduling on Kubernetes · DDP / PyTorch AI training workloads

Slurm 層（namespace: slurm）

  • • slurm-controller：slurmctld + slurmrestd，核心排程器
  • • slurm-login：使用者 sbatch / srun 入口，Lmod 環境
  • • slurmdbd → MySQL：記帳與作業歷史
  • • worker-cpu × 4、worker-gpu-a10/h100 × 4：slurmd 工作節點
  • • DDP AllReduce 流量在 worker 間直接傳遞（net2 subnet）
  • • Secrets：munge-key、ssh-hostkey、jwt-secret

Elastic Operator（自動擴縮）

  • • 每 15 秒查詢 slurmrestd：積壓作業 + 節點狀態
  • • CheckpointAwareQueuePolicy：scale-down 前檢查 checkpoint 時間戳
  • • StatefulSetActuator：透過 K8s API patch StatefulSet replicas
  • • 三個獨立 Pool：cpu / gpu-a10 / gpu-h100
  • • Cooldown 以 Annotation 持久化，pod 重啟不重置
  • • Circuit Breaker：連續失敗後指數退避（最大 60 s）

監控 & 儲存

  • • Prometheus 抓取：slurm-exporter :9341、operator :8000、kube-state-metrics :8080
  • • Grafana :3000：Slurm job、GPU、K8s 資源三合一 Dashboard
  • • Alertmanager :9093：Alert Rules → Email / Slack
  • • NFS PVC (RWX)：controller、worker、login 共享 /shared/jobs（checkpoint）
  • • NVIDIA Device Plugin (kube-system)：暴露 nvidia.com/gpu 資源
  • • NetworkPolicy：namespace 隔離 + 精細 egress/ingress 規則
