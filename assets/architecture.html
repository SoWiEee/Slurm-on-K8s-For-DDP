Phase 6 contributions (thesis core)

  • • M2/M3 lua job_submit: score = α·f_mps_fit + β·f_vram_fit − δ·f_fragmentation + ε·f_runtime_short
  • • M5 runtime-predictor: LightGBM service, lua calls via curl --max-time 200ms; cold-start fallback returns min(user_time_limit, 4h)
  • • M7 reconciler: detects head-pending block + frees low-priority victim; default shadowMode=true after M8 sim showed net-negative on victim-selection-by-priority
  • • Ghost-job detector: emits GhostJobsWedge alert when slurmrestd reports running jobs but no pod is Ready (E7 hardening)

Vanilla Slurm (kept upstream)

  • • slurmctld with priority/multifactor + backfill scheduler on :6817
  • • slurmrestd on :6820 (JWT-HS256) — operator + slurm-exporter both consume
  • • slurmdbd ↔ mysql accounting; lua plugin sends pred_runtime → time_limit when applyTimeLimit=true
  • • slurmd on workers (:6818); step launch over plain TCP; epilog reports back through slurmdbd

Worker pools + autoscaling

  • • cpu: 4 cpu / 3.5Gi mem per pod; partition `cpu`; min=1, autoscaled by operator
  • • gpu-rtx4070: 1×RTX 4070 sliced by NVIDIA device-plugin time-slicing into 2 logical GPUs; slurm.conf advertises 200 mps slots total
  • • gpu-rtx4080: whole-GPU exclusive (no MPS); partition routes from `--constraint gpu-rtx4080`
  • • Cooldown persisted on StatefulSet annotation; ghost-detector guards against scale-up deadlock

Monitoring + reliability

  • • Prometheus scrapes operator (:8000), predictor (:8080), slurm-exporter (:9092)
  • • Alert rules: GhostJobsWedge · ProvisioningLatencyHigh · ScalingFlapping · WorkerPodsStuck · QueueWaitTooLong · OperatorPollSlow
  • • Alertmanager → Grafana annotations; severity routing (critical / warning)
  • • Grafana dashboards mirror operator JSON logs for cold start traceability
